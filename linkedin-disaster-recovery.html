Page 1 — System Overview & Business Continuity Philosophy

What This System Is

A "Business Continuity Orchestrator" that automatically detects regional failures, promotes standby regions, and coordinates data recovery for LinkedIn's critical services. The system must handle full-region loss, maintain data consistency, and orchestrate dependency recovery — all without manual intervention while meeting strict RTO/RPO targets.

  • Zero Manual Intervention — Fully automated detection, failover, and recovery

Disaster Scenarios by Scope

Critical Failure Categories

  • Availability Zone Failure: Single AZ down → Load balancer redirect (30s RTO)
  • Regional Failure: Entire AWS region down → Cross-region failover (5min RTO)
  • Service-Specific Outage: Application failure → Service-level DR (2min RTO)
  • Data Corruption: Bad deployment or data → Point-in-time recovery (15min RTO)
  • Network Partition: Cross-region connectivity loss → Split-brain prevention
  • Cascading Failure: Dependency chain failure → Dependency-aware recovery

Reliability Stressors (What Can Break)

Advanced Failure Modes

  • Split-Brain Failover: Both regions believe they are primary → Data divergence
  • Replication Lag: Data loss > RPO → Inconsistent user experience
  • Flaky Health Probes: False positives triggering unnecessary failover
  • Dependent Service Delay: DB restored before cache → Cascading errors
  • DNS Propagation Delay: TTL causes traffic to failed region
  • Cross-Region Network Partition: Replication broken but regions still accessible
  • Partial Region Degradation: Some services healthy, others failed

Raft-Based Decision Controller

Split-Brain Prevention: Distributed consensus for failover decisions

// LinkedIn DR Decision Controller with Raft consensus
public class DRDecisionController implements RaftStateMachine {
    
    private enum DRState { STABLE, EVALUATING, FAILOVER_INITIATED, FAILOVER_COMPLETE }
    private volatile DRState currentState = DRState.STABLE;
    
    // Raft cluster: 5 nodes across 3 DCs (2 in primary, 2 in standby, 1 in EI4)
    private final RaftGroup raftGroup = RaftGroup.builder()
        .addPeer("ltx1-dr-1", "ltx1-dr-2")  // Primary region
        .addPeer("lva1-dr-1", "lva1-dr-2")  // Standby region  
        .addPeer("ei4-dr-1")                // Neutral tiebreaker
        .build();
    
    // Failover decision requires Raft consensus
    public void evaluateFailover(HealthAssessment assessment) {
        if (currentState != DRState.STABLE) {
            return; // Already in progress
        }
        
        if (assessment.shouldTriggerFailover()) {
            // Propose failover to Raft cluster
            FailoverProposal proposal = FailoverProposal.builder()
                .timestamp(Instant.now())
                .primaryRegion(assessment.getFailedRegion())
                .targetRegion(assessment.getTargetRegion())
                .healthEvidence(assessment.getEvidence())
                .build();
            
            // Submit to Raft for consensus (majority vote required)
            CompletableFuture<Boolean> consensus = raftGroup.propose(proposal);
            
            consensus.thenAccept(approved -> {
                if (approved) {
                    executeFailover(proposal);
                } else {
                    logger.warn("Failover proposal rejected by Raft consensus");
                }
            });
        }
    }
    
    // Raft state machine implementation
    @Override
    public void applyOperation(RaftOperation operation) {
        if (operation instanceof FailoverProposal) {
            FailoverProposal proposal = (FailoverProposal) operation;
            
            // Update state machine
            currentState = DRState.FAILOVER_INITIATED;
            
            // Execute coordinated failover
            orchestrationPipeline.executeFailover(proposal);
            
            // Audit decision
            auditLogger.critical("DR failover initiated via Raft consensus: {}", proposal);
        }
    }
}

Replication Manager Architecture

Snapshot + Stream Pattern: Minimal RPO with efficient bandwidth usage

// Replication strategy for different data stores
public class ReplicationManager {
    
    // Espresso: Async replication with 30s RPO
    public void configureEspressoReplication() {
        EspressoReplicationConfig config = EspressoReplicationConfig.builder()
            .primaryCluster("ltx1-espresso")
            .replicaCluster("lva1-espresso")
            .replicationMode(ASYNC)
            .batchSize(1000)
            .maxLagSeconds(30)
            .snapshotFrequency(Duration.ofHours(6))
            .build();
            
        espressoClient.enableReplication(config);
    }
    
    // Venice: Change capture with RocksDB snapshots
    public void configureVeniceReplication() {
        VeniceReplicationConfig config = VeniceReplicationConfig.builder()
            .primaryStore("ltx1-venice")
            .replicaStore("lva1-venice")
            .changelogTopic("venice-changelog-ltx1-to-lva1")
            .snapshotStrategy(HOURLY_INCREMENTAL)
            .compressionCodec(LZ4)
            .build();
            
        veniceClient.enableReplication(config);
    }
    
    // Kafka: Cross-region mirroring
    public void configureKafkaReplication() {
        MirrorMakerConfig config = MirrorMakerConfig.builder()
            .sourceCluster("ltx1-kafka")
            .targetCluster("lva1-kafka")
            .topicWhitelist("member-events|feed-updates|job-applications")
            .replicationFactor(3)
            .acks("all")
            .enableIdempotence(true)
            .build();
            
        kafkaMirrorMaker.start(config);
    }
    
    // Monitor replication lag across all systems
    @Scheduled(fixedDelay = 15000) // every 15s
    public void monitorReplicationHealth() {
        Map<String, Duration> lagMetrics = Map.of(
            "espresso", espressoClient.getReplicationLag(),
            "venice", veniceClient.getReplicationLag(),  
            "kafka", kafkaClient.getReplicationLag()
        );
        
        for (Map.Entry<String, Duration> entry : lagMetrics.entrySet()) {
            Duration lag = entry.getValue();
            String system = entry.getKey();
            
            metricsClient.gauge("replication.lag.seconds", lag.getSeconds(),
                              Map.of("system", system));
            
            if (lag.getSeconds() > 45) { // Approaching RPO limit
                alertManager.warn("Replication lag high for {}: {}s", system, lag.getSeconds());
            }
        }
    }
}

Split-Brain Prevention with Raft

Distributed Leadership Election

5-node Raft cluster ensures single primary region globally

// Raft consensus for failover decisions
public class DRLeaderElection {
    // 5-node cluster: 2 in LTX1, 2 in LVA1, 1 in EI4
    private final RaftClient raftClient = RaftClient.builder()
        .cluster("dr-coordination")
        .members(List.of("ltx1-dr-1", "ltx1-dr-2", "lva1-dr-1", "lva1-dr-2", "ei4-dr-1"))
        .build();
    
    public CompletableFuture<Boolean> proposeRegionFailover(String from, String to) {
        RegionFailoverCommand cmd = new RegionFailoverCommand(from, to);
        return raftClient.propose(cmd); // Requires 3 of 5 votes
    }
}
